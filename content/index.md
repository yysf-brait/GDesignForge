👋 你好，我是 **Brait**。

欢迎来到 **GDesignForge**，这里是我关于**游戏设计**的个人数字花园。

建立这个站点的初衷，是为了构建一个能够随时记录、整理和分享知识的空间。  
相比于严肃的系统教程，这里更多是我在游戏设计路上的：

- **知识总结**：对游戏机制、数值模型、交互逻辑的拆解与分析。
- **随笔笔记**：碎片化的思考、阅读笔记以及对现有游戏的观察。
- **灵感存档**：那些稍纵即逝的创意火花。

希望这里的某些内容能成为你创作路上的“锻造炉”（Forge），为你提供一份热量或灵感。

同时也非常欢迎你通过评论、邮件或 QQ 等方式与我交流想法和建议，为这个知识花园注入更多活力。

## ⚒️ 参与共建

**GDesignForge** 不仅仅是个人的数字花园，也致力于成为游戏设计知识的公共熔炉。  
如果你有独到的见解、翻译的文章或是发现内容的疏漏，非常欢迎向本站投稿。

### ✍️ 署名须知

无论你是使用何种方式投稿文章或内容，**请务必附带你的署名信息**以保留你的创作权益。

对于**Markdown**格式的投稿文件，请使用 **Metadata** 来显性标识作者信息，示例如下：

```markdown
---
title: 文章标题
author: 你的署名
tags:
  - "u/你的署名"
---
文章正文内容...
```

对于其他文本格式的投稿文件，请在正文中**注明作者署名**。

### 📧 方式一：邮件投稿（推荐）

如果你专注于内容创作，不熟悉代码或 Git 操作，这是最直接的方式。

1. **准备内容**：推荐使用**Markdown**格式编写，当然其他常见文本格式（如 Word）也是可以的。
2. **发送邮件**：请将文件发送至 [brait.w216@gmail.com](mailto:brait.w216@gmail.com)。
3. **人工合并**：收到邮件后，我会人工审核并帮你将内容合并到仓库中。

由于需要人工手动处理，合并与回复可能会有**一定的时间延迟**，请见谅。

> [!warning] 注意
> 通过邮件合并**无法自动生成 GitHub 的贡献记录**  
> 因此请**尤其**注意**附带你的署名信息**以保留你的创作权益。  
> 详见上方的**署名须知**部分。

### 🐙 方式二：GitHub 协作

如果你熟悉 **Git** 工作流（Fork & Pull Request），或者希望对网站代码进行改进，欢迎直接在仓库中提交修改。

我们将采用标准的 **Fork & Pull** 流程来确保协作的顺畅。  
关于详细的**环境配置**、**工作流程**及**提交规范**，请移步至 GitHub 仓库查看完整的开发指南。

👉 **[前往 GitHub 查看协作指南](https://github.com/yysf-brait/GDesignForge#开发协作)**
